xshot 是一款桌面截图工具，支持快捷截图、窗口/区域选择、标注、截长图、裁剪、复制和保存，默认通过托盘和全局快捷键使用。

## 核心能力

- 系统托盘常驻运行，主窗口默认隐藏。
- 全局截图快捷键，默认快捷键：
  - macOS：`Option + X`
  - Windows / Linux：`Alt + X`
- 主窗口可配置快捷键、程序坞图标、开机自启、默认保存位置、界面语言和 macOS 权限入口。
- 支持窗口 hover 识别，移动到候选窗口后可直接点击选中窗口区域。
- 支持手动拖拽框选区域，选区可移动，并可通过边角控制点调整大小。
- 支持手动向下滚动截长图，按选区逐帧捕获、自动拼接并实时预览。
- 长截图生成后可继续裁剪，再复制到剪贴板或保存 PNG。
- 截图工具栏跟随选区显示，并避开顶部安全区域。
- 内置简体中文和英文界面。

## 标注工具

- 序号标注
- 箭头
- 矩形选框
- 直线
- 文字
- 画笔
- 橡皮擦
- 框选马赛克

标注工具的颜色、粗细、字号等属性挂在对应工具按钮的二级面板里。点击已有标注会优先选中它，选中后可以移动，也可以继续通过对应工具面板修改属性；点击选区空白位置才会创建新的标注。

## 使用方式

启动应用后，xshot 会进入托盘运行。你可以通过以下方式开始截图：

- 按默认全局快捷键 `Option + X` / `Alt + X`。
- 点击主窗口中的 `截图` 按钮。
- 点击托盘菜单中的 `Capture`。

截图时：

- 移动鼠标到窗口上，出现候选边框后点击可选中该窗口。
- 拖拽任意区域可手动创建截图选区。
- 选区完成后，工具栏会显示在选区附近。
- 点击标注工具后，在选区空白处创建标注；点击已有标注会优先选中。
- 点击截长图按钮后，选区旁会出现长截图预览浮窗；在选区内向下滚动，停下后会自动采样并拼接。反向滚动和横向滚动会被忽略，避免污染拼接结果。
- 长截图浮窗只显示实时预览和快捷键提示：`Enter` 生成长图，`Esc` 取消。
- 长图生成后会回到编辑视图，选区手柄可用于裁剪长图；复制和下载会导出当前裁剪区域。
- `Enter` 或确认按钮会复制当前截图/裁剪结果到剪贴板。
- 下载按钮会保存 PNG。
- `Esc` 或关闭按钮会取消截图。

## 设置项

- 快捷键：点击编辑后输入新的组合键，保存后立即生效。
- 重置快捷键：恢复默认 `Option + X` / `Alt + X`。
- 程序坞图标：macOS 下可控制是否在 Dock 中显示应用图标。
- 开机自启：登录系统后自动启动 xshot。
- 默认保存位置：下载截图时优先保存到指定文件夹；未设置时使用下载目录。
- 语言：支持简体中文和 English。
- 权限：macOS 下可查看屏幕录制和辅助功能授权状态，并直接打开对应系统设置面板。

## 平台与权限

- macOS 首次截图可能需要授予屏幕录制权限；授权后建议重启应用。
- macOS 截长图需要辅助功能权限，用于监听/过滤滚轮事件并让选区下方窗口接收滚动。
- 程序坞图标开关仅适用于 macOS。
- 当前截图主路径只处理主显示器，多显示器支持仍在完善。
- 窗口 hover 识别依赖系统窗口枚举，部分系统窗口、遮罩层或全屏应用可能表现不同。

## 当前截图链路

- 应用启动时会创建并隐藏截图 WebView，触发截图时复用该窗口。
- macOS 常规截图当前使用系统 `screencapture -x -R <截图窗口区域>`，产物会先写入临时 PNG，再读回前端编辑层。
- Windows / Linux 当前通过 `xcap` 捕获显示器，并在 Rust 侧编码 PNG。
- macOS 截长图会让截图窗口鼠标穿透，滚轮事件只透传向下滚动；每帧优先使用 CoreGraphics `CGWindowListCreateImage` 捕获截图窗口下方的选区内容，失败时回退到 `screencapture -R`。
- 截长图拼接按两帧真实纵向位移追加新增行，小位移不更新上一帧，避免重复纹理或白底导致一次追加过多内容。
- 长图生成后进入裁剪/编辑视图，复制和保存会按当前裁剪区域导出。
- 截图流程保留了分段耗时日志，方便定位快捷键触发、截图获取、图片解码、窗口展示等阶段的延迟。
- 此前试验过 ScreenCaptureKit，但当前质量和收益未达到预期，因此主路径继续保留稳定的 fallback 方案。

## 开发

Tauri 环境安装参考：<https://v2.tauri.app/start/prerequisites/>

前置要求：

- Node.js
- pnpm
- Rust
- Tauri v2 系统依赖

常用命令：

```bash
pnpm install       # 安装依赖
pnpm dev           # 启动 Tauri 开发环境
pnpm dev:web       # 只启动 Vite 前端
pnpm build:web     # 构建前端
pnpm build         # 构建桌面应用
pnpm tsc           # TypeScript 检查
pnpm format        # Prettier + cargo fmt
```

项目结构：

```text
src/                    React 前端
src/windows/            截图窗口
src/logic/              设置、快捷键、光标等前端逻辑
src-tauri/              Tauri / Rust 后端
src-tauri/src/lib.rs    截图、托盘、剪贴板、窗口命令
public/                 应用图片资源
```

## 当前限制

- 多显示器支持仍不完整。
- 截长图目前是 macOS 优先能力，依赖屏幕录制和辅助功能权限；当前只支持向下滚动拼接。
- 标注属性修改目前即时生效，尚未作为独立动作纳入撤销栈。
- 图片格式选择、启动参数、工具栏自定义等高级设置尚未开放。
- 窗口截图依赖候选窗口识别，极少数透明窗口、系统浮层或全屏空间可能无法准确命中。

## 路线图
