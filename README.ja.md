xshot は、クイックキャプチャ、ウィンドウ/範囲選択、注釈、スクロールキャプチャ、トリミング、画面固定、コピー、保存に対応したデスクトップスクリーンショットツールです。通常はトレイとグローバルショートカットから使います。

## 多言語ドキュメント

簡体字中国語の README がドキュメントの基準です。機能説明、インストール時の注意、制限事項、ロードマップを変更するときは、まず `README.md` を更新し、その後 English、繁體中文、日本語、한국어、Español、Français、Deutsch、Português (Brasil)、Русский の各版を同期してください。

## 主な機能

- ✅ スクロールキャプチャに対応。
- ✅ キャプチャ結果を常に最前面のフローティングウィンドウとして固定できます。
- ✅ OCR テキスト認識、QR コード認識、テキスト翻訳、原文上への翻訳オーバーレイに対応。
- ✅ 注釈ツール：番号マーカー、矢印、矩形、直線、テキスト、ペン、消しゴム、範囲モザイク。
- ✅ ウィンドウの hover 検出に対応し、候補ウィンドウ上でクリックするとそのウィンドウを選択できます。
- ✅ 可視ウォーターマークと不可視ウォーターマークに対応。エクスポート時に透明テキストを重ねたり、検出可能な周波数領域ウォーターマークを書き込めます。

## 使い方

起動後、xshot はトレイで動作します。スクリーンショットは次の方法で開始できます：

- デフォルトのグローバルショートカット `Option + X` / `Alt + X` を押す。

## プラットフォーム権限とインストール

- macOS で初回起動時に開発元を検証できないと表示された場合は、`システム設定` -> `プライバシーとセキュリティ` で `このまま開く` を選択してください。
- それでも開けない場合は、`xattr -dr com.apple.quarantine /Applications/xshot.app` を実行してから再試行してください。
- macOS では初回キャプチャ時に画面収録権限が必要になることがあります。許可後はアプリの再起動を推奨します。
- macOS のスクロールキャプチャにはアクセシビリティ権限が必要です。ホイールイベントを監視/フィルタリングし、選択範囲下のウィンドウへスクロールを渡すために使います。
- OCR は macOS Vision を利用します。翻訳にはネットワークアクセスが必要で、デフォルトでは Google Translate を使います。
- Dock アイコンの表示切り替えは macOS 専用です。
- 現在のキャプチャ経路は主ディスプレイのみを対象にしています。マルチモニター対応は改善中です。
- ウィンドウ hover 検出はシステムのウィンドウ列挙に依存するため、一部のシステムウィンドウ、オーバーレイ、フルスクリーンアプリでは挙動が異なる場合があります。

## 設定

- ショートカット：編集をクリックして新しいキー組み合わせを入力し、保存するとすぐに反映されます。
- ショートカットのリセット：デフォルトの `Option + X` / `Alt + X` に戻します。
- Dock アイコン：macOS で Dock にアプリアイコンを表示するかどうかを制御します。
- ログイン時に起動：システムへサインインした後に xshot を自動起動します。
- デフォルトの保存先：ダウンロード時に指定フォルダーへ優先保存します。未設定の場合は Downloads を使います。
- 可視ウォーターマーク：コピー、ダウンロード、画面固定時にカスタム透明テキストを重ねます。四隅、水平タイル、斜めタイルに対応しています。
- 不可視ウォーターマーク：コピー、ダウンロード、画面固定時にカスタム文言を埋め込みます。設定画面で画像を選んで検出でき、検出結果が長く省略された場合だけ hover で全文を表示します。
- インターフェース言語：現在は簡体字中国語と English に対応しています。
- 権限：macOS で画面収録とアクセシビリティの許可状態を確認し、対応するシステム設定を直接開けます。

## 現在のキャプチャ処理

- アプリ起動時にスクリーンショット用 WebView を作成して非表示にし、キャプチャ開始時に再利用します。
- macOS の通常キャプチャは現在、システムの `screencapture -x -R <screenshot-window-rect>` を使います。結果はいったん一時 PNG に書き出し、フロントエンドの編集レイヤーへ読み戻します。
- Windows / Linux では現在 `xcap` でディスプレイをキャプチャし、Rust 側で PNG にエンコードします。
- macOS のスクロールキャプチャではスクリーンショットウィンドウをマウス透過にし、下方向のホイールイベントのみを透過します。各フレームはまず CoreGraphics `CGWindowListCreateImage` でスクリーンショットウィンドウ下の選択範囲を取得し、失敗時は `screencapture -R` にフォールバックします。
- スクロールキャプチャの合成は、2 フレーム間の実際の縦方向移動量から新しい行だけを追加します。小さな移動では前フレームを更新しないため、繰り返し模様や白背景で過剰に追加されるのを避けます。
- 長いスクリーンショット生成後はトリミング/編集ビューに入り、コピーと保存は現在のトリミング範囲をエクスポートします。
- 画面固定では現在のエクスポート PNG を一時ディレクトリに書き出し、境界線なし・常に最前面・全ワークスペース表示の独立した Tauri ウィンドウで画像を表示します。
- ウォーターマークは最終エクスポート時だけ適用され、コピー、ダウンロード、画面固定を対象にします。OCR、QR 認識、翻訳はウォーターマークの干渉を避けるため元の選択範囲を使います。
- 不可視ウォーターマークの主経路は 8x8 DCT の輝度中周波係数ペアへ差分埋め込みを行い、header と本文を冗長に書き込み、検出時は多数決で復元します。payload には magic、長さ、チェックサムを含め、小さい画像や旧エクスポート向けに旧 LSB 経路も互換フォールバックとして残しています。
- OCR は macOS Vision の `VNRecognizeTextRequest` で文字を認識し、accurate を優先して失敗時に fast へフォールバックします。QR コード認識には `VNDetectBarcodesRequest` を使います。
- 翻訳は Rust バックエンドから Google Translate を呼び出し、システムプロキシに対応します。翻訳オーバーレイは OCR block の座標から編集可能かつ取り消し可能なテキスト注釈を生成し、もう一度クリックすると生成済みオーバーレイを削除します。
- キャプチャ処理には段階別の所要時間ログを残しており、ショートカット起動、キャプチャ取得、画像デコード、ウィンドウ表示などの遅延を追跡できます。
- 以前 ScreenCaptureKit を試しましたが、品質と効果が期待に届かなかったため、現在の主経路では安定した fallback 方式を維持しています。

## 開発

Tauri 環境の準備：<https://v2.tauri.app/start/prerequisites/>

必要なもの：

- Node.js
- pnpm
- Rust
- Tauri v2 のシステム依存関係

よく使うコマンド：

```bash
pnpm install       # 依存関係をインストール
pnpm dev           # Tauri 開発環境を起動
pnpm dev:web       # Vite フロントエンドのみ起動
pnpm build:web     # フロントエンドをビルド
pnpm build         # デスクトップアプリをビルド
pnpm tsc           # TypeScript チェック
pnpm format        # Prettier + cargo fmt
```

プロジェクト構成：

```text
src/                    React フロントエンド
src/windows/            スクリーンショットウィンドウ
src/logic/              設定、ショートカット、カーソルなどのフロントエンドロジック
src/logic/watermark.ts  可視ウォーターマーク描画、不可視ウォーターマークの埋め込みと検出
src-tauri/              Tauri / Rust バックエンド
src-tauri/src/lib.rs    キャプチャ、トレイ、クリップボード、ウィンドウコマンド登録
src-tauri/src/ocr.rs    macOS Vision OCR / QR コード認識
src-tauri/src/translation.rs  翻訳サービス
public/                 アプリ画像リソース
```

## 現在の制限

- マルチモニター対応はまだ不完全です。
- スクロールキャプチャは現在 macOS 優先の機能で、画面収録とアクセシビリティ権限に依存します。現時点では下方向の合成のみ対応しています。
- OCR は現在 macOS 優先の機能です。翻訳はネットワークと Google Translate の利用可否に依存します。
- 注釈プロパティの変更はすぐ反映されますが、独立した操作として undo スタックにはまだ入っていません。
- 不可視ウォーターマークは軽量な追跡と検出のためのもので、DRM や改ざん防止ではありません。同一サイズの PNG/JPEG/WebP 再エンコードには旧 LSB より強くなっていますが、大幅なリサイズ、トリミング、回転、強い圧縮、フィルター処理、二次スクリーンショットでは検出できない場合があります。
- 画像形式の選択、起動パラメータ、ツールバーのカスタマイズなどの高度な設定はまだ公開されていません。
- ウィンドウキャプチャは候補ウィンドウ検出に依存するため、ごく一部の透明ウィンドウ、システムオーバーレイ、フルスクリーンスペースでは正確に命中しない場合があります。
