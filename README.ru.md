xshot — это инструмент для скриншотов рабочего стола с быстрым захватом, выбором окна/области, аннотациями, прокручиваемым захватом, обрезкой, закреплением на экране, копированием и сохранением. Он рассчитан на работу из трея и через глобальное сочетание клавиш.

## Основные Возможности

- ✅ Поддерживает прокручиваемый захват.
- ✅ Позволяет закреплять результаты как плавающие окна поверх остальных.
- ✅ Поддерживает OCR, распознавание QR-кодов, перевод текста и наложение перевода поверх исходного текста.
- ✅ Поддерживает инструменты аннотаций: нумерованный маркер, стрелка, прямоугольник, линия, текст, перо, ластик и мозаика области.
- ✅ Поддерживает распознавание окна при наведении: наведите курсор на окно-кандидат и щелкните, чтобы выбрать его.

## Использование

После запуска xshot работает из трея. Начать захват можно так:

- Нажмите глобальное сочетание клавиш по умолчанию `Option + X` / `Alt + X`.

## Разрешения Платформы и Установка

- Если macOS сообщает, что разработчик не может быть проверен, откройте `Системные настройки` -> `Конфиденциальность и безопасность` и выберите `Все равно открыть`.
- Если приложение все еще не открывается, выполните `xattr -dr com.apple.quarantine /Applications/xshot.app` и попробуйте снова.
- На macOS первый захват может потребовать разрешение на запись экрана; после выдачи разрешения рекомендуется перезапустить приложение.
- На macOS прокручиваемый захват требует разрешение Accessibility, чтобы отслеживать/фильтровать события колесика и передавать прокрутку окну под выбранной областью.
- OCR основан на macOS Vision; перевод требует сетевого доступа и по умолчанию использует Google Translate.
- Переключатель значка Dock доступен только на macOS.
- Текущий основной путь захвата работает только с основным дисплеем. Поддержка нескольких мониторов еще улучшается.
- Распознавание окна при наведении зависит от перечисления системных окон, поэтому некоторые системные окна, оверлеи или полноэкранные приложения могут вести себя иначе.

## Настройки

- Сочетание клавиш: нажмите редактировать, введите новую комбинацию и сохраните, чтобы применить ее сразу.
- Сброс сочетания: восстанавливает `Option + X` / `Alt + X`.
- Значок Dock: на macOS управляет отображением значка приложения в Dock.
- Запуск при входе: автоматически запускает xshot после входа в систему.
- Место сохранения по умолчанию: скачиваемые скриншоты сначала сохраняются в указанную папку; если она не задана, используется Downloads.
- Язык интерфейса: сейчас поддерживаются упрощенный китайский и English.
- Разрешения: на macOS можно проверить состояние разрешений записи экрана и Accessibility, а также открыть соответствующую панель системных настроек.

## Текущий Пайплайн Захвата

- При запуске приложение создает и скрывает WebView для скриншотов, а затем переиспользует это окно при начале захвата.
- На macOS обычный захват сейчас использует системную команду `screencapture -x -R <screenshot-window-rect>`; результат сначала записывается во временный PNG, затем загружается в слой редактирования фронтенда.
- На Windows / Linux текущий путь захватывает дисплей через `xcap` и кодирует PNG на стороне Rust.
- На macOS прокручиваемый захват делает окно скриншота прозрачным для мыши и пропускает только события колесика вниз. Каждый кадр сначала пытается захватить содержимое под окном скриншота через CoreGraphics `CGWindowListCreateImage`; при ошибке используется fallback `screencapture -R`.
- Сшивка длинного скриншота добавляет только новые строки по реальному вертикальному смещению между двумя кадрами. Небольшие смещения не обновляют предыдущий кадр, чтобы повторяющиеся текстуры или белый фон не приводили к чрезмерному добавлению контента.
- После генерации длинный скриншот открывается в режиме обрезки/редактирования; копирование и сохранение экспортируют текущую область обрезки.
- Закрепление записывает текущий экспортированный PNG во временный каталог, а затем создает независимое окно Tauri без рамки, поверх остальных и видимое во всех рабочих пространствах.
- OCR использует macOS Vision `VNRecognizeTextRequest`, предпочитает accurate и при ошибке переключается на fast; распознавание QR использует `VNDetectBarcodesRequest`.
- Перевод выполняется Rust-бэкендом через Google Translate и поддерживает системный прокси. Наложение перевода создает редактируемые и отменяемые текстовые аннотации по координатам OCR block; повторный щелчок удаляет созданное наложение.
- В процессе захвата сохраняются логи времени по этапам, чтобы находить задержки при срабатывании сочетания, получении захвата, декодировании изображения и показе окна.
- ScreenCaptureKit ранее тестировался, но качество и выгода не оправдали ожиданий, поэтому основной путь сохраняет стабильный fallback.

## Разработка

Подготовка окружения Tauri: <https://v2.tauri.app/start/prerequisites/>

Требования:

- Node.js
- pnpm
- Rust
- Системные зависимости Tauri v2

Полезные команды:

```bash
pnpm install       # Установить зависимости
pnpm dev           # Запустить режим разработки Tauri
pnpm dev:web       # Запустить только фронтенд Vite
pnpm build:web     # Собрать фронтенд
pnpm build         # Собрать desktop-приложение
pnpm tsc           # Проверка TypeScript
pnpm format        # Prettier + cargo fmt
```

Структура проекта:

```text
src/                    React-фронтенд
src/windows/            Окно скриншота
src/logic/              Настройки, сочетания клавиш, курсор и другая frontend-логика
src-tauri/              Tauri / Rust бэкенд
src-tauri/src/lib.rs    Захват, трей, буфер обмена, регистрация оконных команд
src-tauri/src/ocr.rs    macOS Vision OCR / распознавание QR
src-tauri/src/translation.rs  Сервис перевода
public/                 Ресурсы изображений приложения
```

## Текущие Ограничения

- Поддержка нескольких мониторов все еще неполная.
- Прокручиваемый захват сейчас ориентирован на macOS и зависит от разрешений записи экрана и Accessibility; пока поддерживается только сшивка при прокрутке вниз.
- OCR сейчас ориентирован на macOS; перевод зависит от сети и доступности Google Translate.
- Изменения свойств аннотаций применяются сразу, но пока не учитываются как отдельные действия в стеке отмены.
- Расширенные настройки, такие как выбор формата изображения, параметры запуска и настройка панели инструментов, пока не доступны.
- Захват окна зависит от распознавания окон-кандидатов; редкие прозрачные окна, системные оверлеи или полноэкранные пространства могут не определяться точно.
